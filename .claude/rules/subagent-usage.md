# コードを書く際の Subagent 活用ルール

## 必須: コードを書く前に調査する

**コードを書く前に、必ず Explore subagent で調査すること。**

```
Task(subagent_type: "Explore", prompt: "...")
```

### 調査すべき内容
- 既存の実装パターン
- 関連するファイルの構造
- 使用されているライブラリやユーティリティ
- 類似の実装がないか

## 複雑な実装の前に計画する

3ファイル以上に影響する変更、新機能追加の場合:

```
Task(subagent_type: "Plan", prompt: "...")
```

## 禁止事項

- ❌ 調査なしでコードを書き始める
- ❌ 既存コードを読まずに新規実装する
- ❌ 類似実装を確認せずに新しいパターンを導入する

## 推奨フロー

1. **Explore** で対象コードと周辺を調査
2. 既存パターンを把握
3. 必要に応じて **Plan** で計画
4. コードを書く
